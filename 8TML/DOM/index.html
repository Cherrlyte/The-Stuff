In theory, all non-H3 tags (AKA Children of Body) wield the "Verdana" font, while h3 wields "Comic Sans".
In practice, it works, and i hate it.

How it works:
CSS Styling follows a DOM tree, as in, Document Object Model. The name doesn't matter though, what matters is what it does.
DOM makes it so styling is applied by object, which by default makes it so that EVERY object with specific styling WILL receive such styling.
EXAMPLE: If i have a completely populated body element with multiple other elements (h1, h3, div, i, whatever), ALL elements with their styling defined WILL be styled accordingly. Those who don't have defined styling will, by default, inherit the styling from their parent element .

Think of it like this: The element div doesn't have styling, but the parent element (header) does, which means:
Header ===> Div
But the h3 element inside div does have its own styling, so it does a:
Header ===> Div =X=> H3 (( =X=> as in H3 does not inherit styling, rather uses its own. ))
However, if we give the div elements its own style AND remove h3's...
Header =X=> Div ===> H3 (( =X=> as in Div doesn't inherit styling, rather uses its own. ))
Div doesn't inherit styling, and H3 inherits Div's.

Also, funny thing, it doesn't matter how low in the hierarchy it is, if it's defined, it WILL use styling.
Header ===> Div ===> Div =X=> Article ===> Div ===> Div =X=> Article ===> Div ===> P =X=> Em (( =X=> as in element doesn't inherit styling, since it has its own. ))

Also, notice how EVERY element gets their styling inherited (except for Em, of course). That means you can style one element and ALL ITS CHILDREN WILL BE STYLED LIKE IT. An example is a div with fifteen thousand p elements as children will naturally style all those fifteen thousand elements the same as it is styled.
